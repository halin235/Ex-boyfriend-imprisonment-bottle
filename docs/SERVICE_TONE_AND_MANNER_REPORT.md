# 서비스 톤앤매너 변경 리포트

## 목적

토스 인앱 비게임 미니앱 심사에서 게임으로 오인될 수 있는 표현을 제거하고, 앱 경험을 "감정 마주하기"와 "비워내기" 중심의 힐링 서비스 톤으로 재정리했습니다.

## 핵심 원칙

- 조작/승패/파괴 느낌을 주는 표현을 사용하지 않습니다.
- 마이크는 게임 조작 수단이 아니라 음성 테라피를 위한 입력 도구로 설명합니다.
- 진행 상태는 점수나 레벨이 아니라 감정 정리의 깊이로 표현합니다.
- 완료 화면은 삭제/파괴가 아니라 비워냄과 마무리로 표현합니다.

## 주요 문구 변경

| 기존 톤 | 변경 톤 |
| --- | --- |
| 전남친 봉인 항아리 | 감정 비우기 |
| 봉인 강도 | 비워낸 정도 |
| 미련과 분노를 속삭이며 봉인하라 | 차분히 말하며 감정을 비워내세요 |
| 속삭이기 시작 | 전남친한테 못한 말 쏟아내기 |
| 속삭이기 다시 시작 | 전남친한테 못한 말 다시 쏟아내기 |
| 마이크 끄기 (리소스 해제) | 음성 테라피 멈추기 |
| 봉인이 한계에 달했어요 | 감정을 비워내는 중이에요 |
| 봉인이 곧 풀릴 것 같아 | 감정이 많이 올라왔어요 |
| 작게 속삭일수록 봉인이 빨라져요 | 차분히 말할수록 감정 정리가 깊어져요 |
| 전남친이 저항하고 있어요 | 천천히 낮은 목소리로 이어가 주세요 |
| 항아리에 대고 작게 말해 보세요 | 떠오르는 감정을 편하게 말해 보세요 |
| 지금 나가면 진행 중인 봉인이 저장되지 않아요 | 지금 나가면 진행 중인 감정 정리가 저장되지 않아요 |
| 🗑 삭제 | 비워냄 |

## 마이크 안내 개선

마이크 권한 안내에는 "음성 테라피"라는 사용 목적을 명시했습니다.

- 마이크는 목소리의 크기와 흐름을 감지하는 데만 사용됩니다.
- 사용자가 감정을 말로 마주하고 비워내는 경험을 돕기 위한 기능입니다.
- 대화가 끝나면 마이크가 꺼진다는 점을 안내합니다.
- 권한을 거부해도 서비스를 계속 둘러볼 수 있음을 유지했습니다.

## 수정된 영역

- 앱 표시명: `granite.config.ts`
- HTML title/favicon: `index.html`
- 초기 로딩 화면: `src/components/LoadingScreen.js`
- 메인 진행 문구와 CTA: `src/GameApp.js`
- 진행 상태 라벨: `src/components/GaugeBar.js`
- 음성 상태 안내: `src/components/WhisperStatusBar.js`
- 마이크 권한 안내: `src/components/MicPermissionGate.js`
- 뒤로가기 확인 모달: `src/components/ExitConfirmModal.js`
- 완료 장면 문구/접근성 라벨: `src/components/FinalEndingScene.js`, `src/components/DestroySequence.js`

## 심사 관점 자체 점검

- 화면에 직접 노출되는 게임/레벨/항아리 깨기/파괴/삭제/봉인/저항 문구를 제거했습니다.
- 진행 구조는 남아 있지만, 사용자는 점수나 승패가 아니라 감정 정리 과정으로 인식하게 됩니다.
- 마이크 기능은 조작이 아니라 음성 테라피 목적의 입력으로 설명됩니다.
- 토스 네이티브 내비게이션과 충돌하는 커스텀 상단 바는 이전 작업에서 제거된 상태를 유지했습니다.
