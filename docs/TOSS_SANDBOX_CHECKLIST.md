# 토스 미니앱 전수검사 체크리스트 (업로드 전)

개발자가 샌드박스 업로드 **전** 직접 확인할 항목입니다.

## A. 빌드·배포

- [ ] `npm run build` 성공, `dist/` 생성 확인
- [ ] `granite.config.ts`에 `appName`, `permissions`(microphone) 설정
- [ ] `@apps-in-toss/web-framework` 버전이 콘솔 가이드와 호환되는지 확인
- [ ] HTTPS 또는 샌드박스 허용 URL로 배포 (로컬만 테스트 시 한계 있음)

## B. 뒤로가기 · 종료 (검수 핵심)

- [ ] 물리/앱바 **뒤로가기** → 즉시 종료되지 않고 **「게임이 종료됩니다」** 모달 표시
- [ ] **계속하기** → 게임 유지
- [ ] **나가기** → `closeView()` 호출 (`ExitConfirmModal` → `tryCloseTossView`)
- [ ] `toss.exit()` 가 아닌 **`closeView`** 사용 (웹 미니앱 공식 API)
- [ ] 모달이 열린 상태에서 뒤로가기 한 번 더 → 모달 닫힘 (이중 종료 방지)

## C. 마이크 권한

- [ ] 최초 진입 시 권한 안내 **1회**, 자동 `getUserMedia` 없음
- [ ] **거부** 후 앱 크래시 없음, `MicFallbackBanner` + 설정 가이드 노출
- [ ] 설정 변경 후 **속삭이기 다시 시작**으로 재시도 가능
- [ ] `localStorage`로 모달 반복 노출 방지 동작 확인
- [ ] 마이크 종료/파괴 연출 후 `AudioContext.close()` 및 트랙 `stop()` 확인

## D. UI · Safe Area

- [ ] `viewport-fit=cover` 적용
- [ ] 노치·하단 홈 인디케이터와 버튼/상태바 겹침 없음
- [ ] 로딩 → 게임 전환 시 레이아웃 깨짐 없음

## E. 게임플레이 · 햅틱

- [ ] 에셋 프리로드 완료 후 게임 시작
- [ ] 게이지 25/50/75/100% 햅틱 + 캐릭터 미세 떨림
- [ ] 100% → 봉인 해제 → 파괴 햅틱 + 연출 + 마이크 해제
- [ ] 햅틱 미지원 기기에서도 게임 정상 진행

## F. 성능 · 보안

- [ ] 프로덕션 빌드에 `console.log` 없음
- [ ] API 키·토큰·개인정보 하드코딩 없음
- [ ] 백그라운드 전환 시 마이크 분석 중단 (`visibilitychange`)

## G. 샌드박스 실기기 시나리오 (권장)

1. 신규 설치 → 권한 허용 → 속삭임 → 게이지 상승 → 봉인 해제
2. 신규 설치 → 권한 거부 → 설정 가이드 확인 → 설정에서 허용 → 재시도
3. 게임 중 뒤로가기 → 모달 → 나가기 → 미니앱 종료
4. 게임 중 홈/다른 앱 전환 후 복귀 → 마이크·UI 정상
